Cryptography

The mathematics of secrets: keeping information safe in an open world.

What Is Cryptography?

Cryptography transforms readable information into scrambled nonsense that only the intended recipient can unscramble. It's been used for millennia—from ancient Roman military codes to the encrypted messages on your phone right now.

Security Through Mathematics

Modern cryptography doesn't rely on keeping the method secret. The algorithms are public—anyone can see how AES or RSA works. Security comes from mathematical problems that are easy to create but impossibly hard to reverse without the key.

Every time you see that padlock icon in your browser, cryptography is at work. Your passwords, bank transactions, private messages—all protected by mathematical transformations that would take billions of years to crack.

Caesar Cipher

One of the oldest encryption methods: shift each letter by a fixed amount. Julius Caesar reportedly used a shift of 3. It's trivially breakable today (only 26 possible keys!), but it introduces the core concept: a reversible transformation controlled by a key.

The Key Distribution Problem

Classical encryption has a fundamental problem: both parties need the same secret key. How do you share a key securely? If you already had a secure channel, you wouldn't need encryption! This is the key distribution problem.

In 1976, Whitfield Diffie and Martin Hellman proposed a revolutionary solution: public key cryptography. You have two keys—a public key you share with everyone, and a private key you keep secret. Anyone can encrypt with your public key, but only you can decrypt with your private key.

The Trapdoor

Public key systems rely on "trapdoor" functions: easy to compute one way, hard to reverse. Multiplying two large primes is easy; factoring their product is hard. This asymmetry is what makes RSA secure—encryption is easy, but breaking it would require factoring a number with hundreds of digits.

Hash Functions

A hash function takes any input and produces a fixed-size output (the "hash" or "digest"). It's a one-way function—you can't reverse it to find the input. Even tiny changes to the input completely change the hash. This is used for password storage, file verification, and digital signatures.

7f83b1657ff1fc...

Try changing just one character!

"Hello, World!" 7f83b1657ff1fc...
"Hello, World?" 1d91ca4c1e7b9a...

What hashes are used for:

  • Passwords: Websites store hashes, not your actual password
  • File integrity: Verify downloads haven't been tampered with
  • Digital signatures: Sign the hash instead of the whole document
  • Blockchain: Each block contains the hash of the previous block

Cryptography in Action

HTTPS: When you see the padlock, your browser and the server have performed a "handshake" using public key crypto to establish a shared secret, then switched to faster symmetric encryption for the actual data.

End-to-End Encryption: Apps like Signal encrypt messages so that even the company running the servers can't read them. Only the sender and recipient have the keys.

Digital Signatures: Just as you can encrypt with a public key and decrypt with a private key, you can sign with a private key and verify with a public key. This proves a message came from you and wasn't altered.

The Quantum Threat

Quantum computers could break RSA by efficiently factoring large numbers. That's why cryptographers are developing "post-quantum" algorithms that will remain secure even against quantum attacks. The race is on to deploy them before large quantum computers arrive.

Key Insights

Security is mathematical. Modern encryption doesn't rely on obscurity. The algorithms are public; security comes from mathematical hardness. If a cipher is only secure because the method is secret, it's not truly secure.

Public key crypto solved key distribution. The ability to encrypt to someone without first sharing a secret revolutionized secure communication. It's why you can securely connect to a website you've never visited before.

Hashing is everywhere. One-way functions are fundamental to security. Your passwords aren't stored as passwords. File downloads are verified with hashes. Blockchains are chains of hashes.

Where This Applies

Web: HTTPS, SSL/TLS certificates. Communication: Encrypted messaging, email security. Finance: Online banking, cryptocurrency. Authentication: Passwords, digital signatures, certificates.
